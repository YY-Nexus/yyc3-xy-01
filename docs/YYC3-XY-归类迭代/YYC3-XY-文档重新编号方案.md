# YYC3-XY 文档重新编号方案

## 一、编号原则

### 1.1 核心原则

- **唯一性**：每个编号只能对应一个文档
- **有序性**：编号体现文档的逻辑顺序和上下文关系
- **可扩展性**：预留编号空间，便于后续添加新文档
- **标准化**：遵循YYC3文档命名规范

### 1.2 编号结构

```
[类别编号][序号]-YYC3-XY-[文档类型]-[具体名称].md
```

- **类别编号**：1位数字，表示文档所属大类
- **序号**：2位数字，表示文档在该类别中的顺序
- **文档类型**：技巧类、架构类、规范类、实施类、审核类、规划类、用户类

## 四、编号实施计划

### 4.1 实施步骤

1. **准备阶段**
   - 备份所有文档
   - 创建编号映射表
   - 准备重命名脚本

2. **执行阶段**
   - 按类别顺序重命名文档
   - 更新文档内的交叉引用
   - 验证编号唯一性

3. **验证阶段**
   - 检查所有文档编号
   - 验证文档可访问性
   - 更新文档索引

### 4.2 时间安排

| 阶段 | 任务 | 预计时间 | 负责人 |
|-----|------|---------|--------|
| 准备阶段 | 备份文档、创建映射表 | 30分钟 | 文档管理员 |
| 执行阶段 | 重命名文档 | 2小时 | 文档管理员 |
| 执行阶段 | 更新交叉引用 | 1小时 | 文档管理员 |
| 验证阶段 | 验证编号唯一性 | 30分钟 | 文档管理员 |
| 验证阶段 | 更新文档索引 | 30分钟 | 文档管理员 |

**总计**：4.5小时

## 五、风险评估与缓解

### 5.1 风险识别

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|------|------|---------|
| 重命名导致链接失效 | 高 | 中 | 批量更新所有引用 |
| 文档丢失 | 高 | 低 | 完整备份后再操作 |
| 工作量超预期 | 中 | 中 | 分批处理，优先处理重要文档 |
| 团队协作冲突 | 中 | 低 | 提前沟通，制定计划 |

### 5.2 回滚方案

如果重命名过程中出现问题，执行以下回滚步骤：

1. 停止重命名操作
2. 从备份恢复文档
3. 通知团队成员
4. 分析问题原因
5. 制定修复方案

## 六、后续维护

### 6.1 编号管理规范

1. **编号分配原则**
   - 每个编号只能对应一个文档
   - 编号分配后不得随意更改
   - 新文档按顺序分配编号
   - 废弃文档保留编号并标记状态

2. **编号冲突处理**
   - 发现冲突立即整改
   - 优先保留最新或最重要的文档编号
   - 其他文档重新分配编号

3. **编号维护机制**
   - 建立编号分配记录表
   - 定期检查编号唯一性
   - 文档重命名需更新索引

### 6.2 定期检查

- **月度检查**：检查编号唯一性
- **季度检查**：检查文档格式规范性
- **年度检查**：全面审核文档体系

### 7.2 重命名脚本示例

```bash
#!/bin/bash
# 文档重命名脚本

# 定义映射关系
declare -A map=(
    ["01-YYC3-XY-规范类-COPPA合规实施指南.md"]="101-YYC3-XY-规范类-COPPA合规实施指南.md"
    ["02-YYC3-XY-规范类-标准化检查清单.md"]="102-YYC3-XY-规范类-标准化检查清单.md"
    # ... 更多映射关系
)

# 执行重命名
for old_name in "${!map[@]}"; do
    new_name="${map[$old_name]}"
    find /Users/yanyu/yyc3-小语/docs -name "$old_name" -type f -exec mv {} {}_temp \;
    find /Users/yanyu/yyc3-小语/docs -name "${old_name}_temp" -type f -exec bash -c 'mv "$0" "${0%/*}/'"$new_name"'"' {} \;
done
```
